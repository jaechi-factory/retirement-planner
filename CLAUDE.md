# retirement-planner 작업 규칙

## 저장소
- 대상 저장소: https://github.com/jaechi-factory/retirement-planner

## 제품 원칙
- 이 프로젝트는 비개발자도 이해 가능한 은퇴 계산기여야 한다.
- 복잡한 금융 구조를 설명하더라도 결과는 직관적으로 보여줘야 한다.
- 사용자가 먼저 이해해야 할 것은 계산식이 아니라 판정과 의미다.

## 커뮤니케이션 원칙
- 중간 구현 설명은 최소화한다.
- 최종 결과 보고는 아래 순서를 따른다.
  1. 최종 판정
  2. 사용자 관점에서 바뀐 점
  3. 확인한 시나리오
  4. 남은 리스크
  5. 다음 액션
- 코드 중심 설명보다 사용자 영향 중심으로 설명한다.

## 자동차 기능 원칙
- 자동차는 투자자산이 아니라 사용 자산 + 지속 비용으로 본다.
- 차 1대만 지원한다.
- 구매 / 할부 구매 / 리스·렌트 / 이미 보유를 구분한다.
- 월 유지비, 남은 잔액, 이율, 기간, 종료 시 처분가/반환금을 반영한다.
- 생활비와 자동차 비용 중복 계산을 반드시 방지한다.
- 결과는 아래 3가지를 최우선으로 보여준다.
  - 자동차 포함 시 가능 생활비
  - 자동차 제외 시 가능 생활비
  - 자동차로 인해 줄어드는 금액

## 우선순위
- 기능의 정확성 > 설명의 그럴듯함
- 사용자 이해도 > 모델링의 과도한 정교함
- MVP 출시 가능성 > 기능 과확장
